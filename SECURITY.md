# Security Policy

## Reporting a vulnerability

Please **do not** open a public issue for anything security-sensitive (authentication bypass, data exposure, injection, privilege escalation, etc.).

Instead, use GitHub's private vulnerability reporting for this repository:

**[Report a vulnerability](https://github.com/worksquares/datasquares-issues/security/advisories/new)**

Include as much detail as you can: affected platform, steps to reproduce, and impact. We will acknowledge your report, investigate, and keep you updated until the issue is resolved. Please allow us reasonable time to remediate before any public disclosure.

## Scope

This policy covers the DataSquares product on all platforms. For issues in other WorkSquares products, use the corresponding product's issue repository.
